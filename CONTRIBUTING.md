# Contributing to PerC AI

Thanks for your interest in contributing! Follow these guidelines to keep the project consistent.

- Fork the repository and create a feature branch: `git checkout -b feature/your-feature`
- Keep commits focused and atomic. Use descriptive commit messages.
- Run linters and tests (frontend lint available).
- Open a Pull Request against `main` describing the change, rationale, and any manual steps to verify.

Coding style
- Follow existing project conventions: React + functional components for frontend, FastAPI idioms for backend.
- Keep UI changes accessible and responsive.

Communication
- For larger changes, open an issue first and discuss the design.

License
- By contributing you agree your contributions will be licensed under the project's MIT License.
